## Cross-Phase Integration (Integration Checker Output)

**Wiring:** 8 connections traced, 0 orphaned, 0 missing.

### Key integration points verified

1. **Dual-route handler wiring (ROUTE-01..05).** Both `Routes()` and `NextcloudRoutes()` in `web/webdav/webdav.go` iterate the same `webdavMethods` slice and share `resolveWebDAVAuth` via an `authed` group. Mounted at `/dav` and `/remote.php` respectively in `web/routing.go:270/276`. Custom HTTP methods (MKCOL, COPY, MOVE, PROPPATCH) registered explicitly on the top-level Echo instance at `routing.go:361-363` (required because `echo.Any()` doesn't include them).

2. **Cross-phase auth coverage (AUTH-01..05).** Phase 1's `resolveWebDAVAuth` is the sole auth gate for all Phase 2 verbs (PUT, DELETE, MKCOL, MOVE) and all Phase 3 verbs (COPY, PROPPATCH). No verb bypasses it on either route.

3. **MOVE → dead property store (MOVE-01..05 ↔ Phase 3 `deadprops.go`).** `move.go:132` calls `deadPropStore.movePropsForPath` — this is the one explicit cross-phase handler call. Phase 2 handler invokes Phase 3 state. Verified by litmus `propmove` test (PASS).

4. **End-to-end flows.** Three integration tests cover the full session:
   - `TestE2E_GowebdavClient` (Phase 1) — connect, PROPFIND, GET, OPTIONS, HEAD on both routes
   - `TestE2E_WriteOperations` (Phase 2) — PUT, MKCOL, MOVE, DELETE
   - `SuccessCriterion6_Copy` (Phase 3) — file + directory COPY via real gowebdav client
   All pass against a live test instance.

5. **Litmus compliance across all phases (TEST-06).** `scripts/webdav-litmus.sh` runs against both routes:
   - basic 16/16 ✓ (touches Phase 1 READ + Phase 2 PUT/DELETE/MKCOL)
   - copymove 13/13 ✓ (touches Phase 2 MOVE + Phase 3 COPY)
   - props 30/30 ✓ (touches Phase 1 PROPFIND + Phase 3 PROPPATCH/deadprops)
   - http 4/4 ✓ (touches Phase 2 PUT Expect:100-continue)

6. **Documentation completeness (DOC-01..04).** `docs/webdav.md` (587 lines after audit-phase updates) documents all 10 implemented methods with curl examples, client config, and now includes a "Client compatibility with `/remote.php/webdav/`" section capturing the manual validation finding.
