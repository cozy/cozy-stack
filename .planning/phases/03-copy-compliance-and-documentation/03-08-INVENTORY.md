## Issues Discovered and Fixed

### Issue 1: JWT token too long for litmus Basic Auth (>256 chars)

**Symptom:** `init` test FAILED with "password must be <256 chars"

**Cause:** The litmus `neon` library enforces a 256-character limit on HTTP
Basic Auth passwords. The JWT token generated by `cozy-stack instances token-cli`
using a long timestamped domain (e.g., `litmus-20260412-171209.localhost:8080`) was
262 characters — over the limit.

**Fix:** Changed `scripts/webdav-litmus.sh` to use a shorter domain name format:
- Before: `litmus-YYYYMMDD-HHMMSS.localhost:8080` (~37 chars)
- After: `lm-EPOCH.localhost:8080` (~25 chars)

The shorter domain reduces the JWT payload `iss` claim length, bringing the token
to ~244-248 characters (well under 256).

**File:** `scripts/webdav-litmus.sh`

### Issue 2: MKCOL/COPY/MOVE return 405 Method Not Allowed

**Symptom:** `begin` test FAILED with "MKCOL returns 405 Method Not Allowed"

**Cause:** The front-end Echo instance in `CreateSubdomainProxy` uses `main.Any("/*", ...)`
to route all requests to `firstRouting()`. Echo's `Any()` only registers standard
HTTP methods plus PROPFIND and REPORT (which have explicit slots in Echo's router).
Custom WebDAV methods MKCOL, COPY, and MOVE were not in Echo's `methods` array and
therefore fell through to Echo's built-in 405 handler before reaching `firstRouting()`.

**Fix:** Added explicit route registrations for MKCOL, COPY, MOVE (and PROPPATCH) in
`CreateSubdomainProxy` in `web/routing.go`:

```go
for _, m := range []string{"MKCOL", "COPY", "MOVE", "PROPPATCH"} {
    main.Add(m, "/*", firstRouting(router, appsHandler))
}
```

**File:** `web/routing.go`

### Issue 3: Incomplete propfind.go (unused imports, missing WithPrefix functions)

**Symptom:** `go build` failed with unused imports and undefined functions.

**Cause:** Prior work on plan 03-07 (props suite) left `propfind.go` in an intermediate
state — the `handlePropfind` function was updated to call `buildResponseForDirWithPrefix`,
`buildResponseForFileWithPrefix`, and `streamChildrenWithPrefix`, but those functions
were not yet fully added to the file (the `WithPrefix` helpers were partially there,
`streamChildrenWithPrefix` was added, and the old `buildResponse*` functions now delegate
to the new ones).

**Fix:** Completed the implementation — all `*WithPrefix` functions are defined, imports
are correct, and the old non-prefix functions now delegate to the prefix versions using
`davFilesPrefix` as the default.

**File:** `web/webdav/propfind.go`

### Issue 4: Duplicate handleProppatch / proppatch.go already existed

**Symptom:** `handleProppatch redeclared in this block`

**Cause:** `web/webdav/proppatch.go` already contained a full RFC 4918-compliant
PROPPATCH handler (Strategy B: 207 rejecting all dead properties with 403). When the
linter added PROPPATCH to the dispatcher in `handlers.go`, and a stub was briefly added,
there were two declarations.

**Fix:** Removed the stub — `proppatch.go` already provided the correct implementation.
