<objective>
Fix the pre-existing FOLLOWUP-01 test-harness race so that `go test -race -count=1 ./web/webdav/...` runs with zero DATA RACE warnings. The race is between `pkg/config.UseViper` (called in test setup of test N) and the long-lived `AntivirusTrigger` goroutine spawned by `stack.Start` in test N-1 which reads `config.FsURL()` on a timer.

Approach (per CONTEXT.md preference ladder — Option A first):
- Gate the registration of `AntivirusTrigger` in `memScheduler.StartScheduler` (and the Redis scheduler equivalent) on an env var `COZY_DISABLE_AV_TRIGGER`.
- In `web/webdav/testutil_test.go`, set that env var (via `t.Setenv`) before any call that triggers `stack.Start` (specifically before `setup.GetTestInstance()` which drives `stack.Start` → `StartScheduler`).
- If Option A leaves the race fully closed, ship. If Option A still shows DATA RACE reports after application (because another goroutine not related to antivirus triggers the same read path), escalate to Option B (add `stack.Shutdown`-equivalent in `t.Cleanup`).

Purpose: DEBT-01 closed; every subsequent Phase 4-7 test can run with `-race` cleanly, which is a prerequisite for the LARGE + CONC tests in Phases 5 and 7.
Output: gated registration, env-var set in test harness, DATA RACE reports eliminated.
</objective>
